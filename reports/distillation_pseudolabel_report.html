BirdCLEF 2026 的核心挑戰是 訓練域 (train_audio) → 測試域 (soundscape) 的 domain shift。 Teacher-Student Distillation 與 Pseudo Labeling 是兩種主流解法：

3.2 PCEN Frontend（可學習）

class AudioToMelPCEN(nn.Module):
    # 每個 mel bin 一組可學習參數 (shape: 1 × 128 × 1)
    alpha_raw = nn.Parameter(torch.full(shape, logit(0.98)))   # smoothing
    delta_raw = nn.Parameter(torch.full(shape, softplus_inv(2.0)))  # bias
    r_raw     = nn.Parameter(torch.full(shape, logit(0.5)))    # root
    s_raw     = nn.Parameter(torch.full(shape, logit(0.25)))   # IIR decay

    # PCEN 公式：
    # M = IIR_smooth(S, s)  (background estimate)
    # PCEN = (S / (ε + M)^α + δ)^r − δ^r
PCEN vs AmplitudeToDB：PCEN 做 自適應增益正規化，動態壓縮背景噪音， 對 soundscape 的靜音段和強噪段都有更好的抗干擾效果。

3.3 Loss Function（Masked BCE）

class MaskedBCEWithLogitsLoss(nn.Module):
    def forward(self, logits, primary_targets, secondary_targets):
        combined = torch.clamp(primary_targets + secondary_targets, 0.0, 1.0)
        raw_loss = BCE_with_logits(logits, combined)

        # 遮罩 secondary label 的梯度，只保留 primary label 監督
        mask = torch.ones_like(raw_loss)
        mask[secondary_targets == 1.0] = 0.0
        mask[primary_targets  == 1.0] = 1.0

        return (raw_loss * mask).mean()

四、Embedding Distillation（表示學習蒸餾）

來源：train_embed_distill.py（本地 codebase）

4.1 核心思想

不蒸餾分類結果，而是蒸餾 Perch v2 的 1536-dim embedding 空間。 Student 骨幹（EfficientNet-B0）在音訊 mel 上提取 embedding， 訓練目標是讓其 L2-normalized embedding 與 Perch teacher 對齊。

Student 架構：
  EfficientNet-B0
    → GEMFreqPool (p=3.0, freq 維度 pooling)
    → AdaptiveAvgPool1d (time 維度 pooling)
    → Linear(feat_dim) → GELU → Dropout
    → Linear(→ 1536) → LayerNorm → L2-normalize
    → unit-sphere embedding (1536-dim)

4.2 Loss Functions

# 方法 1：Cosine Loss（默認）
loss = (1.0 - (student_emb * teacher_emb_norm).sum(dim=-1)).mean()
# 兩端都 L2-normalize，最大化餘弦相似度

# 方法 2：InfoNCE / NT-Xent（v3 config）
def infonce_loss(student_emb, teacher_emb, temperature=0.1):
    B = student_emb.shape[0]
    sim = torch.matmul(student_emb, teacher_emb.T) / temperature  # (B, B)
    labels = torch.arange(B, device=student_emb.device)
    # 對角線 = 正例對，非對角線 = batch 負例
    return (F.cross_entropy(sim, labels) + F.cross_entropy(sim.T, labels)) / 2

六、三種方法比較與建議

方法Teacher資料優點缺點建議
Noisy Student 5-fold SED ensemble 未標記 soundscape Domain adaptation 最強；可迭代提升 需要先有好的 SED 模型 優先嘗試
Embedding Distill Perch v2 (frozen) 所有 train_audio + soundscape 骨幹學到豐富音訊表示；無標籤也能用 需要先 cache Perch embeddings 優先嘗試
Output Distill Perch label-head 已標記 soundscape 直接對齊分類目標；實作簡單 限於 Perch 的 234 類空間 次要

建議實驗路線

  1. Embedding Distillation：用 train_embed_distill.py (configs v4 or v5)， 在 train_audio + soundscapes 上蒸餾 Perch embedding。cosine loss 或 InfoNCE 都可以試。 完成後骨幹接 ProtoSSM 或分類頭 fine-tune。
  2. Pseudo Label 生成：用當前最好的 model（Perch probe）對未標記 soundscape 推理， 套用 Power Transform (γ=2.0) + 95th percentile 過濾，生成偽標籤 CSV。
  3. Noisy Student 訓練：將偽標籤 CSV 餵進 SED/ProtoSSM 訓練， pseudo_label_weight=0.5 縮小影響，避免 diversity collapse。
注意（已知限制）：根據之前實驗記錄，pseudo labels 在 SED 訓練中會破壞 Perch ensemble 多樣性， 導致整體 ensemble LB 下降。目前只有 ProtoSSM 適合接受偽標籤輔助（因為它不是 Perch 的直接學生）。
