§2 4-Fold Cross-Validation + Model Soup

Why 4-Fold CV Matters (Mathematical Argument)

With a single train/val split, our model is trained on ~80% of data (28,439 files) and validated on 20%. After selecting the best model, we use it for inference — but it has never seen the 20% held-out files.

With 4-fold CV + ensemble:

  • Each fold trains on 75% of data (26,662 files), marginally less per model
  • But 4 models are ensembled — equivalent to ∞ training data via averaging
  • Final ensemble has seen 100% of training data across all folds
  • Model diversity from different train/val splits reduces correlated errors
Var(ensemble) = Var(single) / N × (1 + (N-1)ρ) For N=4 folds, ρ≈0.6 (typical inter-fold correlation): Var(ensemble) ≈ Var(single) × 0.4 StdErr(ensemble) ≈ StdErr(single) × 0.63 Expected AUC gain ≈ 0.63 × (overfit gap) ≈ +0.005~0.012 LB

§3 Post-Processing Deep Dive

We have 8 post-processing functions already implemented in postproc_eval_v2.py. This section provides rigorous analysis of each, including 2025 evidence and recommended parameters.

Test Soundscape Structure

Test soundscapes: 5-second windows, 5-second step (non-overlapping). Each file produces a sequence of predictions. Adjacent windows share no audio — so temporal smoothing is a prior, not a signal.

The key insight from 2025: if a species is present in one window, it's likely present in adjacent windows too (birds don't vanish instantly). Smoothing exploits this temporal autocorrelation.

PP Function Analysis Table

Function2025 UsageMechanismBest ParamsExpected LB ΔRisk
pp_sliding_max(w, window) 1st, 2nd place Blend current clip (weight w) with max of ±window/2 neighbors. Propagates confident detections to adjacent frames. w=0.7, window=3 (covers ±1 clip = ±5s) +0.002~0.006 Low — physically grounded
pp_power_boost(top_n, boost_gamma, suppress_gamma) 3rd place 2026 Boost top-N confident predictions (p^γ, γ<1 amplifies), suppress rest (p^γ, γ>1 suppresses). Sharpens confident detections. top_n=3, boost=0.5, suppress=2.0 +0.002~0.008 Medium — can overfit public LB
pp_file_max_pool 2nd place 2025 Replace each clip's prediction with file-level maximum. Extreme version: assume if species detected once in file, it's present throughout. — +0.000~0.005 High — destroys temporal resolution, assume always-present
pp_blend_vlom(alpha) 2nd place 2025 VLOM = (geometric mean + RMS) / 2. More robust than arithmetic mean for ensemble blending. Handles distribution mismatch between models. alpha=0.5 (50% VLOM, 50% arithmetic) +0.001~0.004 Low — standard ensemble technique
pp_quantile_mix(alpha) 38th place 2025 Blend raw scores with rank-normalized scores within each file. Corrects for systematic over/under-confidence per file. alpha=0.7 (70% raw, 30% rank) +0.001~0.003 Low — smooth interpolation
pp_sigmoid_temperature(T) Standard calibration Temperature scaling: logit/T before sigmoid. T>1 → softens; T<1 → sharpens. Global calibration of confidence. T=0.8~1.2 (tune on val) +0.001~0.003 Low — well-understood
pp_gaussian(sigma) — Gaussian temporal smoothing along time axis. Softer than sliding_max; doesn't propagate peaks. sigma=1.0 +0.000~0.003 Low
pp_threshold_zero(threshold) — Hard-zero predictions below threshold. Suppresses noise floor, improves precision at low recall. threshold=0.01~0.05 +0.000~0.002 Medium — hard cutoff

New Post-Processing: TTA (Already Implemented)

Test-Time Augmentation via eval_sed_holdout_tta.py

Two independent TTA modes:

  • Hop 2.5s: Slide window every 2.5s instead of 5s → 2× denser temporal sampling → average overlapping predictions
  • Circular shift ×3: Shift audio by 0, -2.5s, +2.5s → 3× augmented inference → average
  • Both (mode=both): 2 hops × 3 shifts = 6 inferences averaged per original window

TTA is free signal — no extra training. Expected: +0.002~0.005 LB

New Post-Processing: Perch Calibration

Species-level Platt Scaling (Proposed)

Our Perch predictions are strong but some species are systematically biased. Per-species sigmoid calibration:

p_cal(c) = sigmoid( a_c × logit(p_raw(c)) + b_c ) where (a_c, b_c) fit on validation soundscape predictions vs ground truth

Risk: only 66 soundscape files for calibration → easy to overfit. Use cross-validation within the 13 val files. Expected: +0.001~0.004 LB if calibration is species-specific but regularized.

Ensemble Blending Formula

# Step 1: Within each model type, average fold predictions sed_b0 = mean( fold0_soup, fold1_soup, fold2_soup, fold3_soup ) [B0 ensemble] sed_v2s = mean( vfold0_soup, vfold1_soup, vfold2_soup, vfold3_soup ) [V2S ensemble] perch = weighted_mean( label_pseudo×3.0, ss_train×1.5, embed×1.0 ) / 5.5 # Step 2: VLOM blend of heterogeneous types sed_ensemble = VLOM( sed_b0, sed_v2s, alpha=0.5 ) # geometric-RMS blend # Step 3: Final weighted blend final = (sed_ensemble × w_sed + perch × w_perch) / (w_sed + w_perch) # where w_sed, w_perch tuned on val soundscapes (start: 0.5 / 0.5) # Step 4: Post-processing final = pp_sliding_max(final, w=0.7, window=3) final = pp_sigmoid_temperature(final, T=best_T)

Why VLOM vs Simple Average?

VLOM = (geometric_mean + RMS) / 2 from 2025 2nd place. Rationale:

  • Geometric mean: requires ALL models to agree on a high score → high precision, conservative
  • RMS (quadratic mean): dominated by the strongest signal → high recall, aggressive
  • VLOM = average of both: balances precision and recall, empirically best for AUC metric
  • For SED × Perch blend, models have very different score distributions — VLOM is more robust than arithmetic mean
