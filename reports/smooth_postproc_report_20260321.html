Key Findings

🏆 MaxDilation (radius=1) is the key ingredient that beats Gaussian.
所有超越 Gaussian 的方法都包含 max_dilation。BirdCLEF 2024 第4名已驗證此技術 (+~0.01 LB)。 原理：鳥在多個相鄰 clip 中叫聲但模型只在其中一個 clip 偵測到 → max pool 將此偵測擴散到鄰近 clip → 提高 recall → macro AUC 上升。
✅ Gauss→Dilation 最佳組合 (mean=0.7782, +0.0033)：
先 Gaussian 平滑消除噪聲，再 max_dilation(r=1) 填補偵測間隙。 兩步互補：Gaussian 降低假陽性雜訊；dilation 補回被 Gaussian 截斷的真陽性。
⚠️ 所有可學習方法均輸給 Gaussian。
根本原因：資料量太小（708 windows → 531 training OOF）。 Conv (1170 params)、MultiScale (6318 params) → 過擬合，AUC 崩到 0.695。 SoftOrderStats 勉強追平 (0.7748 vs 0.7749)，因為它學出的 kernel 形狀就是 Gaussian。
❌ 新 audio-domain filter 全部落後： CausalIIR (0.7530), DerivativeOnset (0.7305), SoftTopHat (0.7345)。 原因：這些 filter 假設 Perch 輸出有 onset/offset 結構，但 Perch 機率序列已經平滑過， 沒有明顯的音訊邊緣可以利用。
📊 Median→Gaussian (DCASE 2020-2024 標準) 幾乎追平 Gaussian (+0.0004)。
DCASE 事件偵測 (event F1) 中 median pre-filter 效益顯著 (+9.1pp)，但 BirdCLEF 的 macro AUC 指標對此較不敏感，因為模型輸出已無明顯衝刺雜訊。
📊 HMM forward-backward (-0.0162) 和 MultiScaleMedian (-0.0163) 明顯落後。
HMM 的 p_onset=0.2, p_persist=0.7 是針對短事件設計的參數，BirdCLEF 5s clip 行為不同。 MultiScaleMedian 的 +21.7% event-F1 是 DCASE SED 評估結果，BirdCLEF AUC 目標不一致。

Implementation — 加入 Kaggle 提交 Notebook

在 gauss_smooth_logits() 之後插入以下程式碼（位置：predict_file() 內的 _logits_to_sigmoid）：

# 在 gauss_smooth_logits 之後插入： from scipy.ndimage import maximum_filter1d def apply_dilation_logits(logits, radius=1, T=1.15): """Gauss→Dilation: 填補相鄰 clip 偵測間隙，提高 recall → AUC。 BirdCLEF 2024 4th place 同款技術。radius=1 → 3-clip window。 """ n_files = logits.shape[0] // N_WINDOWS X = logits.reshape(n_files, N_WINDOWS, logits.shape[1]) # Convert to prob space for max-pool (preserves ordering) p = 1.0 / (1.0 + np.exp(-np.clip(X / T, -88, 88))) p_dil = maximum_filter1d(p, size=2*radius+1, axis=1, mode='nearest') p_dil = np.clip(p_dil, 1e-6, 1-1e-6) # Back to logit space (* T so downstream sigmoid(x/T) is correct) return (np.log(p_dil / (1 - p_dil)) * T).reshape(-1, logits.shape[1]).astype(np.float32) # In predict_file(), after gauss_smooth_logits: if GAUSSIAN_SMOOTH and GAUSSIAN_ON_LOGITS: _logits_to_sigmoid = gauss_smooth_logits(_logits_to_sigmoid) _logits_to_sigmoid = apply_dilation_logits(_logits_to_sigmoid) # ← NEW
Config flag 建議： 加入 DILATION_AFTER_GAUSS = True，方便 ablation。 OOF 驗證 lift: +0.0033 macro AUC (59 soundscape files, 4-fold)。 預估 LB impact: 若 OOF→LB 轉換比例類似 Gaussian (+0.010 LB)， 則 dilation 可能帶來 +0.001~0.003 LB。

為什麼 MaxDilation 對 BirdCLEF 有效？

問題：偵測間隙 (Detection Gap)

鳥類鳴叫有時跨越多個相鄰 5s clip，但模型只在其中一個 clip 輸出高機率：

clip: [t1] [t2] [t3] [t4] [t5]
truth: [ 0 ] [ 1 ] [ 1 ] [ 1 ] [ 0 ] ← bird vocalizing
model: [ 0 ] [0.1] [0.8] [0.2] [ 0 ] ← only t3 detected
dil.: [ 0 ] [0.8] [0.8] [0.8] [0.2] ← gap filled ✓

max_dilation(radius=1) 對 t2 取 max(t1,t2,t3)=0.8，對 t4 取 max(t3,t4,t5)=0.8。 Ground truth 在 t2、t4 都有正例 → recall 提升 → AUC 上升。

為什麼 Gaussian 做不到？

Gaussian 是加權平均，會降低峰值：

model: [ 0 ] [0.1] [0.8] [0.2] [ 0 ]
gauss: [0.0] [0.3] [0.5] [0.3] [0.1] ← peak 0.8→0.5
dil: [ 0 ] [0.8] [0.8] [0.8] [0.2] ← peak preserved

Gaussian 將峰值從 0.8 壓低到 0.5，同時也「平均」到非鳥鳴區域。 Dilation 保留最高峰值並傳播到鄰近，不降低 true positive 的置信度。

Gauss→Dilation 組合：Gaussian 先消除單點假陽性（impulse noise）， 再 dilation 填補真陽性的偵測間隙 → 兩步互補。
