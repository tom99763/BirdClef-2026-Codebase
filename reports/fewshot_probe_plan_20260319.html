實作計劃 (Tier 1 → 先做)

Tier 1A — Free improvement

L2-Normalization Before PCA (SimpleShot insight)

Current pipeline uses StandardScaler (unit variance) but no L2-normalization. SimpleShot shows L2-norm + centering outperforms StandardScaler by 3–5% for cosine-based classifiers.

# BEFORE (current)
emb_scaled = StandardScaler().fit_transform(emb)
Z = PCA(64).fit_transform(emb_scaled)

# AFTER (SimpleShot-style)
emb_l2 = emb / (np.linalg.norm(emb, axis=1, keepdims=True) + 1e-8)  # L2-norm
emb_centered = emb_l2 - emb_l2[labeled_mask].mean(axis=0)           # center
Z = PCA(128, whiten=True).fit_transform(emb_centered)                # more dims + whiten

Expected gain: +0.003–0.008 holdout AUC · Effort: 30 min

Tier 1B — Handles rare species

Prototype Fallback for Classes with 2–7 Positives

Currently, classes with n_pos < 8 are skipped entirely (no probe). SimpleShot prototypical classification works reliably from even 2 positives.

# For classes with 2 ≤ n_pos < min_pos_logr: use prototype instead of LogReg
for cls_idx in range(NUM_CLASSES):
    n_pos = y_tr[:, cls_idx].sum()
    if n_pos >= 8:
        # Existing LogReg probe (unchanged)
        clf.fit(X_tr_cls, y_tr[:, cls_idx])
        pred = clf.decision_function(X_va_cls)
    elif n_pos >= 2:
        # Prototype fallback: cosine distance to mean positive
        proto = Z_tr[y_tr[:, cls_idx] == 1].mean(axis=0)
        proto = proto / (np.linalg.norm(proto) + 1e-8)
        pred = Z_va @ proto  # cosine similarity score
    else:
        # 0–1 positives: fall back to Perch native logit only
        continue

Expected gain: +0.002–0.005 (covers ~30 additional rare species) · Effort: 45 min

Tier 1C — Key insight from Tip-Adapter

Support Cache Retrieval (Tip-Adapter Style)

Build a key-value cache from all labeled soundscape clips. At inference, each query clip retrieves its nearest neighbors in embedding space and votes for their labels. Blend with Perch native logits.

# Build cache: keys=embeddings, values=labels
keys_norm = emb_labeled / (np.linalg.norm(emb_labeled, axis=1, keepdims=True) + 1e-8)
# keys_norm: (N_labeled, 1536), values: (N_labeled, 234)

def tip_adapter_score(query_emb, keys, values, tau=0.5):
    q_norm = query_emb / (np.linalg.norm(query_emb, axis=1, keepdims=True) + 1e-8)
    sim = q_norm @ keys.T / tau       # (12, N_labeled)
    weights = np.exp(sim - sim.max(axis=1, keepdims=True))  # stable softmax
    weights /= weights.sum(axis=1, keepdims=True)
    return weights @ values           # (12, 234)

# Blend: alpha*retrieved + (1-alpha)*perch_native_prob
retrieved = tip_adapter_score(emb_12, keys_norm, Y_labeled)
final = alpha * retrieved + (1 - alpha) * perch_native_prob

Expected gain: +0.005–0.015 · Effort: 2 hours · Tune: alpha ∈ [0.2,0.6], tau ∈ [0.1,1.0] via OOF

實作計劃 (Tier 2 — 次要)

Tier 2A

Embedding Graph Smoothing (LaplacianShot-lite)

Replace current 1D temporal smooth (Amphibia/Insecta only) with a 2D embedding-similarity graph over all 12 clips per file. Acoustically similar clips share label information.

def emb_graph_smooth(probs_12, emb_12, alpha=0.3):
    en = emb_12 / (np.linalg.norm(emb_12, axis=1, keepdims=True)+1e-8)
    W = np.maximum(en @ en.T, 0)   # (12,12) cosine
    W /= W.sum(axis=1, keepdims=True)+1e-8
    return (1-alpha)*probs_12 + alpha*(W @ probs_12)

Expected gain: +0.002–0.005

Tier 2B

LP++ Init: Prototype-Seeded LogReg

Initialize LogReg coefficients from class prototype (mean positive embedding) instead of random. Converges faster to a better solution with few positives.

def lpp_logreg(X_tr, y_tr, C=0.5):
    proto = X_tr[y_tr==1].mean(axis=0)
    clf = LogisticRegression(C=C, max_iter=400)
    clf.fit(X_tr, y_tr)
    # warm-start from prototype
    clf.coef_ = proto[None, :]
    clf.fit(X_tr, y_tr)  # refine
    return clf

Expected gain: +0.002–0.004 for rare species

核心理論洞見

為什麼 L2-norm 比 StandardScaler 好?

StandardScaler 讓每個維度方差=1，但不保證 embedding 向量的長度一致。兩個語義相似的 clip 可能因為音量不同而 L2 距離很大。

L2-normalization 後，所有向量落在單位超球面上，cosine distance = Euclidean distance，讓 KNN 和 prototype 距離更有意義。

SimpleShot 在 miniImageNet 上測試：L2-norm+center 比 StandardScaler 高 5–8%。

為什麼 Tip-Adapter 特別適合 BirdCLEF?

我們有 ~2232 個已標記的 soundscape clips 作為 support set。Perch embedding 把同物種的 clip 聚集在同一個 cluster。

Tip-Adapter 的 retrieval 直接利用這個 cluster 結構：test clip 只需要找到幾個鄰居，讓鄰居投票。

Perch 的原始 logit 作為 "zero-shot prior" 正好符合 Tip-Adapter 的框架。兩個資訊源互補。

Transductive 的優勢：每個 soundscape file = 12個 query clips

每個 60s soundscape 裡的 12 個 5s clips 共享同一個錄音環境（背景噪音、混響、設備響應）。在 embedding 空間中，這 12 個向量會形成一個緊密的 cluster，彼此方向相近。

LaplacianShot 的 graph smoothing 利用這個結構：如果 clip A 和 clip B 在 embedding 空間中很近，它們的物種標籤應該相似。這比固定 1D 時序平滑更靈活，因為它同時考慮了時序和聲學相似性。
