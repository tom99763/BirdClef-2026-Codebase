📋1. Executive Summary

The cluster-based stacker proposes a non-parametric, locality-sensitive post-processing layer for the BirdCLEF 2026 ensemble. Rather than learning a global mapping from 1170-dim prediction concatenation to target labels (as MLP/TCN stackers do), it uses HDBSCAN to discover acoustic scene clusters in prediction space and then blends each window's base prediction with the cluster's empirical label distribution. The key novelty is a score-weighted MST distance that inflates pairwise distances for windows where the 5 base models disagree—encoding uncertainty directly into the clustering geometry.

Five experiments (A1–B2) span three axes of variation: feature space (raw scores vs. Perch embeddings vs. combined), distance metric (cosine vs. score-weighted), and training data (labeled only vs. +5K pseudo anchors). Based on theoretical analysis, experiment B2 (Emb+Scores, score-weighted distance, +5K pseudo) has the highest ceiling but also the greatest risk from pseudo-label noise. Experiment A3 is the recommended first-run candidate due to information richness without pseudo-label contamination. Traditional stacker baselines (MLP OOF 0.9594, TCN 0.9007) represent the upper and lower bounds of comparison.

ℹ

This method targets a potential +0.002–0.005 LB improvement over the current 0.929 if cluster structure is genuine and pseudo-label noise is controlled. The primary risk is data sparsity (708 rows in 1170-dim space) causing HDBSCAN to classify most points as noise.

Blend Formula

\[ \hat{y}_i = (1 - \alpha \cdot s_i)\,\cdot\, \text{base}_i \;+\; (\alpha \cdot s_i)\,\cdot\, \mu_{c(i)} \]

where \(\alpha = 0.4\) (blend strength), \(s_i \in [0,1]\) (soft cluster membership), \(c(i)\) = assigned cluster, \(\mu_{c(i)}\) = mean label distribution of cluster \(c(i)\), and \(\text{base}_i = \frac{1}{5}\sum_{k} \sigma(z_{ik})\).

Implementation Pseudocode (Python) 
import numpy as np
import hdbscan

class ClusterStacker:
    def __init__(self, alpha=0.4, beta=2.0, min_cluster_size=10):
        self.alpha = alpha
        self.beta = beta
        self.min_cluster_size = min_cluster_size

    def _score_weighted_dist(self, X, scores):
        # X: (N, D) features; scores: (N, 5, 234) per-model sigmoid outputs
        var_x = np.mean(np.var(scores, axis=1), axis=-1)  # (N,)
        # Compute pairwise cosine distance
        X_norm = X / (np.linalg.norm(X, axis=1, keepdims=True) + 1e-8)
        cos_sim = X_norm @ X_norm.T
        cos_dist = 1.0 - np.clip(cos_sim, -1, 1)
        # Inflate by inter-model variance
        var_i = var_x[:, None]  # column
        var_j = var_x[None, :]  # row
        weight = 1.0 + self.beta * (var_i + var_j)
        return cos_dist * weight

    def fit(self, X, scores, labels):
        # X: feature matrix, scores: (N,5,234), labels: (N,234) binary
        D = self._score_weighted_dist(X, scores)
        self.clusterer = hdbscan.HDBSCAN(
            metric='precomputed',
            min_cluster_size=self.min_cluster_size,
            prediction_data=True
        ).fit(D)
        # Compute per-cluster mean label distribution
        self.cluster_means = {}
        for cid in set(self.clusterer.labels_) - {-1}:
            mask = self.clusterer.labels_ == cid
            self.cluster_means[cid] = labels[mask].mean(axis=0)
        self.X_train = X
        self.train_scores = scores
        return self

    def predict(self, X_test, test_scores):
        base_pred = np.mean(test_scores, axis=1)  # (N,234)
        labels_, strengths_ = hdbscan.approximate_predict(
            self.clusterer, X_test
        )
        preds = base_pred.copy()
        for i, (cid, s) in enumerate(zip(labels_, strengths_)):
            if cid != -1 and cid in self.cluster_means:
                mu = self.cluster_means[cid]
                blend = self.alpha * s
                preds[i] = (1 - blend) * base_pred[i] + blend * mu
        return preds

📐3. Score-Weighted Distance Deep Dive

Motivation

Standard HDBSCAN with cosine distance treats all windows equally—two windows with similar prediction vectors are considered "close" regardless of how confident the models are. But in BirdCLEF, there is a crucial asymmetry: a window where all 5 models strongly agree is qualitatively different from one where they disagree. The score-weighted distance encodes this epistemic uncertainty directly into the clustering geometry.

Mathematical Definition

\[ \text{var}_i = \frac{1}{C}\sum_{c=1}^{C}\,\mathrm{Var}_{k}\bigl[\sigma(z_{ikc})\bigr] \]

Mean per-class variance across the 5 models (\(k=1\ldots5\)), averaged over all \(C=234\) classes.

\[ d_{\mathrm{sw}}(x_i,\, x_j) \;=\; d_{\cos}(x_i,\, x_j)\;\cdot\;\Bigl(1 + \beta\,(\mathrm{var}_i + \mathrm{var}_j)\Bigr) \]

\(\beta = 2.0\) is the uncertainty inflation coefficient. \(d_{\cos} \in [0,2]\) for unit-normalized vectors.

Effect on Clustering

Confident Windows (var ≈ 0)

All 5 models agree on the prediction. Weight factor ≈ 1.0. Distance is essentially pure cosine. These windows form tight, stable clusters—ideal for computing reliable cluster means.

Uncertain Windows (var → high)

Models strongly disagree. Weight factor inflates distance. Even if the prediction vector looks similar to another window, they are kept farther apart. These windows tend to become noise points (label = −1, strength = 0), falling back to base_pred—appropriate behavior.

Connection to Existing Literature

  • UMAP fuzzy simplicial complex: UMAP also modifies edge weights in a KNN graph based on local density. Score-weighted distance is analogous but uses model disagreement as the density signal.
  • Density-sensitive distance: Bijral et al. (2011) showed that density-sensitive geodesic distances improve spectral clustering. Here, density in "model agreement space" modulates the MST.
  • Conformal prediction regions: High inter-model variance correlates with wide conformal prediction sets—windows that are harder to classify. Pushing them toward noise is a principled choice.
  • HDBSCAN mutual reachability: The standard mutual reachability distance already has a built-in density correction (\(\mathrm{max}(d_{\mathrm{core}}(x_i), d_{\mathrm{core}}(x_j), d(x_i,x_j))\)). Score-weighted distance adds a second layer of uncertainty correction before mutual reachability is computed.
Worked Example: β sensitivity analysis

Suppose two windows have cosine distance 0.05 (very similar prediction vectors). Window i has var = 0.001 (confident); window j has var = 0.08 (uncertain).

With 708 rows in 1170-dimensional space, the samples-to-dimensions ratio is 0.61—well below the rule-of-thumb of 5–10 per dimension for stable distance estimation. This is the curse of dimensionality in its most severe form.

⚖6. Theoretical Strengths & Weaknesses

Strengths

Acoustic Scene Structure

Soundscape recordings from the same site/habitat have consistent species compositions. HDBSCAN should discover these "habitat clusters." Windows from a cloud forest will cluster together and share rare highland species detections.

Temporal Coherence

12 consecutive windows from the same 60-second file will have nearly identical prediction vectors. HDBSCAN will naturally group them, providing implicit temporal smoothing without the artifacts of sliding-window smoothing.

Rare Species Benefit

For species with few training examples, cluster means provide a contextual prior: "windows from similar acoustic contexts tend to contain species X." This is analogous to prototype networks (Snell 2017) but applied to the stacking layer.

Noise Robustness

HDBSCAN's noise classification is a feature here: uncertain or atypical windows (strength=0) revert entirely to base prediction. The method degrades gracefully to the base ensemble when clustering fails.

Weaknesses

Data Sparsity

708 points in 1170+ dimensions may yield mostly noise points. If min_cluster_size=10, and HDBSCAN produces only 5–10 clusters, most of the 708 rows will be noise → blend has no effect on most test windows.

!

Mitigation: reduce min_cluster_size (risk: unstable clusters), or use PCA/UMAP to reduce dimensions before clustering (risk: information loss).

Label Imbalance

Cluster means are dominated by zeros (pos_rate=0.018). The blend formula pulls predictions toward near-zero means, potentially hurting recall for rare species that are correctly predicted by the base ensemble.

SED Leakage Distortion

sed_csebbs achieves 0.9957 OOF due to train_audio leakage. Its contribution to the 1170-dim feature space dominates cluster formation for training data but may not generalize, creating train/test cluster mismatch.

OOF Reliability

With only 59 soundscapes, each cross-validation fold contains ~12 files. OOF AUC variance is ±0.015 or more. Hyperparameter selection (α, β, min_cluster_size) based on OOF may not transfer to LB.

🏷7. Pseudo Labels Analysis

What the Pseudo Labels Are

The pseudo labels come from the SED ensemble applied to 10,658 unlabeled soundscapes. They are continuous probabilities (not binary), representing model confidence rather than ground truth. The 5,000 selected anchors are the highest-confidence windows (likely threshold ≥ 0.5 for at least one species).

When Pseudo Labels Help

  • Sparse regions: Habitats not well-represented in 59 labeled files get anchors from the 10K pseudo pool → test windows in those habitats find neighbors.
  • Cluster stability: More points → HDBSCAN can use smaller min_cluster_size → finer-grained clusters with more stable means.
  • Rare species: Pseudo-labeled windows where rare species were confidently detected add rare-species signal to cluster means.

When Pseudo Labels Hurt

  • Label noise propagation: A false positive in pseudo labels (species X detected but not present) will contaminate the cluster mean, biasing all cluster members toward species X.
  • Distribution mismatch: Pseudo labels are continuous (prob=0.6) while true labels are binary (0/1). Cluster means from mixed data have different scale than pure labeled data means.
  • Cluster contamination: A habitat cluster with 2 labeled + 50 pseudo windows will have its mean dominated by pseudo predictions → cluster mean reliability degrades.

Pseudo Label Quality Estimation

The pseudo labels derive from the SED ensemble, which achieves ~0.929 LB AUC on soundscapes. At a 0.5 threshold, expected precision is approximately 60–70% for common species and 40–55% for rare species. Key risks:

Common species pseudo precision (estimated)~65%
Rare species pseudo precision (estimated)~45%
Pseudo labels from SED (same source as sed_csebbs feature)Correlated bias
⚠

Circular dependency risk: The sed_csebbs feature (from the SED ensemble) and the pseudo labels (from the SED ensemble) are correlated. Clustering on sed_csebbs features and then using SED-derived pseudo labels to compute cluster means introduces a circular signal that may artificially inflate OOF while not generalizing to LB.

Recommended Pseudo Label Strategy

  • Run A3 first (no pseudo) to establish a clean baseline cluster quality estimate.
  • If A3 produces ≥30% non-noise points, proceed to B2 with pseudo labels.
  • If A3 produces <10% non-noise points, the labeled data is too sparse; pseudo labels are essential (run B1 next).
  • Consider using only high-confidence pseudo windows (top 1% per species by model agreement) rather than top 5K by any species.

🔍8. HDBSCAN Internals in This Context

How HDBSCAN Builds Clusters

HDBSCAN operates in four stages. Understanding each stage helps predict behavior on this dataset.

Stage 1: Core Distance

For each point \(x_i\) and parameter min_samples \(k\):

\[d_{\text{core},k}(x_i) = \text{distance to the } k\text{-th nearest neighbor of } x_i\]

In this context: With 708 points in 1170-dim space and k=5, core distances will be large (sparse space). Points in dense file-level groups (12 windows/file) will have small core distances; isolated windows will have large core distances.

Stage 2: Mutual Reachability Distance

\[d_{\text{mreach},k}(x_i, x_j) = \max\!\left(d_{\text{core},k}(x_i),\; d_{\text{core},k}(x_j),\; d_{\text{sw}}(x_i, x_j)\right)\]

In this context: For score-weighted distance (Exp A3, B2), \(d_{\text{sw}}\) replaces the standard distance. The max operation means even if two windows have low \(d_{\text{sw}}\), if either has a large core distance (isolated in space), their mutual reachability distance stays large—they won't cluster.

Stage 3: Minimum Spanning Tree

HDBSCAN builds the MST over all pairwise mutual reachability distances. The score-weighted distance modification affects every edge in this MST—uncertain windows will have inflated edges to all their neighbors, making them less likely to be included in dense subgraphs.

Expected MST structure for 708 labeled rows:

  • 12 short edges per file (consecutive windows in same file)
  • Longer edges connecting files from same habitat type
  • Very long edges connecting different habitat types

Stage 4: Condensed Cluster Tree → Soft Membership

The condensed tree is pruned to select persistent clusters (those that survive over a range of density thresholds). Each point receives a soft membership \(s_i \in [0,1]\) equal to its probability of cluster membership normalized by the maximum within-cluster membership.

Key insight: \(s_i = 0\) means the point is a noise outlier → base_pred unchanged. \(s_i = 1\) means the point is a core cluster member → maximum blending toward cluster mean. This is the mechanism that makes the method robust: uncertain windows self-select to \(s_i \approx 0\).

Expected Cluster Count Estimates

With 59 soundscape files × 12 windows and typical habitat diversity in BirdCLEF, we expect:

HDBSCAN approximate_predict for Test Data

After fitting on training data, hdbscan.approximate_predict(clusterer, X_test) assigns each test point to the nearest cluster without re-running the full clustering algorithm. This is critical for efficiency and consistency.

Limitation: The approximation may not find the optimal cluster assignment for test points in ambiguous regions. Test points near cluster boundaries will receive lower membership strength (s close to 0), which is appropriate—they should not be strongly blended.

# Under the hood: approximate_predict
# For each test point x:
#   1. Find nearest training exemplar by core distance
#   2. Assign to that exemplar's cluster
#   3. Membership strength = min(1, exemplar_strength × distance_ratio)
# This is O(N_train × D) per test point → scales with feature dimension

🚀10. Next Steps & Ablations

Recommended Execution Order

  1. Run A1 first (simplest, fastest) to diagnose cluster quality: count clusters, noise%, and mean cluster size.
  2. Inspect cluster structure: If ≥ 12 clusters found with <50% noise, proceed. If mostly noise, consider dimension reduction first.
  3. Run A3 (score-weighted + combined features) as primary candidate.
  4. Run B2 (full version with pseudo) only if A3 shows promise (>0 LB delta).
  5. Skip A2 and B1 unless A3 underperforms A1 (suggests Perch embeddings are hurting).

Alternative Directions if Cluster Stacker Underperforms

K-Nearest Neighbors Blend

Instead of cluster means, use KNN label means from the k=20 nearest labeled windows. More fine-grained than cluster means; no dependency on HDBSCAN quality.

Gaussian Process Stacker

Model label distribution as a GP in prediction space. Uncertainty-aware, naturally handles sparse data, interpolates smoothly. Expensive to compute for 234 species.

Transductive Label Propagation

Build a KNN graph on training + test features, then propagate labels from labeled nodes. Standard semi-supervised approach; well-understood behavior.

File-Level MLP

Train an MLP on file-level aggregated features (mean/max/std across 12 windows), then use predictions to re-weight per-window predictions. Captures temporal context explicitly.

Integration with Existing Pipeline

ℹ

The cluster stacker should be applied as a post-processing layer on top of the MLP stacker output, not as a replacement. Pipeline: base models → MLP stacker (current best 0.9594 OOF) → cluster stacker → final prediction. This ensures the cluster stacker can only improve upon an already-strong baseline, not introduce regressions.

Summary

The cluster-based stacker is a theoretically motivated method that could provide meaningful improvement by exploiting acoustic scene structure and temporal coherence. The key innovation—score-weighted MST distance—is principled and analogous to established uncertainty-aware clustering methods. The primary risk is data sparsity (708 rows in 1170+ dimensions), which may cause HDBSCAN to classify most points as noise, leaving predictions unchanged.

Recommended first run: A3 (2706-dim combined features, score-weighted distance, no pseudo labels) with min_cluster_size=8, α=0.4, β=2.0. Expected runtime: ~15 minutes including OOF evaluation. If A3 cluster noise rate <50%, proceed to B2 with pseudo labels.
