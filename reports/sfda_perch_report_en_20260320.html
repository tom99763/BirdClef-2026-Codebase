1 What is Source-Free Domain Adaptation (SFDA)?

The Problem with Standard Domain Adaptation

Classical Unsupervised Domain Adaptation (UDA) requires simultaneous access to both source and target domain data for joint alignment. In practice, the source data is often unavailable due to privacy regulations, storage costs, IP restrictions, or because only a model checkpoint was shared — not the original training set.

The SFDA Solution

SFDA uses only a pre-trained source model and unlabeled target data — no source samples required. The goal is to adapt the model's representations to fit the target distribution while preserving the knowledge already encoded in the source hypothesis.

Formal definition: Given a source-trained model fs and unlabeled target data Xt, learn an adapted model ft that performs better on the target distribution Pt, without accessing any source domain samples Xs.

UDA (Unsupervised DA)

Requires source data + unlabeled target data. Joint feature alignment during training. Cannot be applied when source data is unavailable.

SFDA (Source-Free DA)

Requires source model + unlabeled target data only. No access to source samples. The most applicable framework for this competition.

Semi-supervised DA

Source model + a small set of labeled target data + unlabeled target data. We have 1,478 labeled soundscape clips — making this an even stronger setting than pure SFDA.

Four co-existing dimensions of domain shift:
1. Geographic Shift

Source data spans the entire globe, while all target soundscapes come exclusively from the Pantanal wetlands of Brazil. The acoustic background — frog choruses, insect swarms, tropical rain — is fundamentally different from any northern-hemisphere training recording.

2. Device Shift

Source clips were recorded with directional microphones by birders targeting a single species. Target clips come from passive Autonomous Recording Units (ARUs) with omnidirectional pickups, producing a different frequency response, gain curve, and signal-to-noise ratio even for the same species.

3. Label / Prior Shift

Source clips have a roughly uniform class prior (one species, intentionally selected). Target clips reflect the true Pantanal species prior — common Pantanal species dominate, many source species are absent. The multi-label co-occurrence structure is also entirely new.

4. Task Shift

Source training is effectively single-label classification (which bird is this?). Target evaluation is multi-label detection (which birds are present in this 5-second window?). The model must simultaneously recognize multiple overlapping calls against a high-noise floor.

Domain Gap in Practice — Same SED model evaluated on source OOF vs. target soundscape val

Holdout AUC is measured on source-domain (train_audio) out-of-fold clips. Soundscape val AUC is measured on target-domain (Pantanal ARU) clips. The ~0.15–0.22 gap directly quantifies domain shift.

3 Why BirdCLEF 2026 Is a Strong Fit for SFDA

✓ Condition 1: Pre-trained source model available

Perch is a bioacoustic foundation model from Google, trained on the full XenoCanto corpus (EfficientNet-B1, 1536-dim embeddings). It serves as a strong source hypothesis. We access it only as a frozen feature extractor — no source training data required.

✓ Condition 2: Target domain data is accessible

10,658 target-domain soundscape files (unlabeled) plus 1,478 labeled target clips from train_soundscapes_labels.csv. Most SFDA benchmarks only provide unlabeled target data — we have both labeled and unlabeled, making our setting strictly stronger.

✓ Condition 3: Train and test soundscapes share the same distribution

Test soundscapes are confirmed to come from the same Pantanal deployment (filename pattern BC2026_Test_*_S05_*). Any adaptation performed on train_soundscapes directly improves test performance, with no risk of distribution mismatch.

✓ Condition 4: Perch embeddings are cross-domain by design

The Perch paper (Nature Scientific Reports, 2023) demonstrated zero-shot transfer to bats, marine mammals, and mosquitoes — all untrained taxa — using a linear probe on frozen embeddings. This strong cross-domain generalization makes Perch an ideal backbone for SFDA in this setting.

Key insight: This is semi-supervised domain adaptation, not pure SFDA.
We have 1,478 labeled target clips in addition to unlabeled soundscapes. This gives us three complementary information sources to exploit simultaneously:

Source model (Perch) → encodes global bioacoustic knowledge  +  Labeled soundscape clips → supervised target-domain signal  +  Unlabeled soundscapes → unsupervised target-distribution knowledge
4 SFDA Methods — Survey and Applicability
3 TENT — Fully Test-Time Adaptation by Entropy Minimization
ICLR 2021 Online BN params only
Core idea: During inference, update only the affine parameters (γ, β) of Batch Normalization layers by minimizing prediction entropy on each incoming batch.

L_TENT = 𝔼[H(f(x))] = −𝔼[Σ_c p_c · log p_c]

Intuition: More confident predictions correspond to lower entropy. Minimizing entropy pushes the model to produce sharper, more decisive outputs on target data — without any labels.

BirdCLEF caveat: Naive entropy minimization collapses in multi-label settings (drives all class probabilities toward zero). A per-class binary entropy formulation is required, combined with a strict bound on the number of update steps.
Implementation effort: Low Risk: Medium (collapse) Expected gain: +1–2pp OOF
2 SHOT — Source Hypothesis Transfer
ICML 2020 Feature-level Pseudo-label
Core idea: Freeze the classifier head (the "source hypothesis") and adapt only the feature extractor using two complementary objectives:

① Information Maximization (IM):
L_IM = −H(p̄) + λ · 𝔼[H(p)]
— H(p̄): maximize marginal entropy (encourage diverse predictions across the batch)
— 𝔼[H(p)]: minimize per-sample entropy (encourage confident per-clip predictions)

② Self-supervised pseudo-labeling: Cluster target embeddings; assign pseudo-labels to high-confidence predictions; retrain the feature extractor iteratively.

Application to Perch: Freeze the Perch backbone. Adapt only the MLP probe head on unlabeled soundscape clips using the IM loss at a very low learning rate.
Implementation effort: Medium Risk: Medium Expected gain: +2–3pp OOF
1 LaplacianShot — Transductive Inference via kNN Graph Already Implemented
ICML 2020 Transductive kNN graph
Core idea: Instead of adapting the model during training, jointly optimize predictions over the entire inference batch by solving an energy minimization problem over a k-nearest-neighbor graph in embedding space:

E = Σ_i H(p_i) + λ · Σ_{i,j∈kNN} ‖p_i − p_j‖²

— First term: each clip's prediction should be confident (low entropy)
— Second term: neighboring clips should produce consistent predictions (local smoothness)

BirdCLEF advantage: Pantanal soundscapes recorded at the same site and time period are acoustically correlated — the same bird calling nearby will appear in several consecutive clips. The kNN graph naturally captures this co-occurrence structure and propagates confident predictions to their neighbors.
Implementation effort: Already done Risk: Low Expected gain: +1–3pp OOF
4 NRC — Neighborhood Reciprocal Clustering
NeurIPS 2021 Graph-based Reciprocal kNN
Core idea: Weight edges in the kNN graph by reciprocity. If A is in B's k-NN and B is in A's k-NN, they are reciprocal neighbors and receive a higher affinity score. One-directional neighbors receive lower weight. A self-regularization loss enforces consistency with this weighted neighborhood consensus.

Motivation: Noisy pseudo-labels from unreliable neighbors degrade adaptation. Reciprocal neighbors are more reliable because the agreement is mutual — treating them with higher confidence reduces noise propagation.

Application to Perch: Build a kNN graph over all soundscape clip embeddings. Use reciprocal consensus to refine probe predictions before final scoring.
Implementation effort: High Risk: Medium Expected gain: +1–2pp OOF
1 FeCAM — Feature Covariance-Aware Mahalanobis NCM Already Implemented
NeurIPS 2023 Prototype-based Covariance-aware
Core idea: Replace the Euclidean distance used in standard nearest-class mean (NCM) prototypical classification with a per-class Mahalanobis distance, estimating a separate covariance matrix for each class using Ledoit-Wolf shrinkage and Tukey regularization.

Why this matters: Different bird species produce embedding clusters with very different shapes — some elongated (calls vary in pitch), some compact (species with stereotyped vocalizations). Euclidean prototypes assume spherical clusters and introduce systematic bias. Mahalanobis distance adapts to the true cluster geometry.

Domain adaptation angle: Covariances can be computed separately on train_audio and soundscape embeddings, enabling domain-aware distance estimation that accounts for distribution shift.
Implementation effort: Already done Risk: Low Expected gain: +2–4pp OOF

Addition 1 — ss_repeat=5
Tile the 739 labeled soundscape clips 5× before concatenating with the 85K train_audio set. This raises the target-domain fraction from 0.86% to 4.2% of each training epoch, giving the MLP 5× more gradient signal from the target distribution without any extra data collection.
Addition 2 — LaplacianShot at inference
After the MLP produces initial logits, run a joint energy minimization over all soundscape clips in the validation/test batch using a k=7 kNN graph. No training required — this is a post-processing step that enforces local prediction consistency purely at inference time.

6 Domain-Aware PCA: The Most Impactful SFDA Technique We Already Use
The most important realization: We have been doing SFDA without explicitly naming it. The configuration pca_fit_splits: [train, soundscape] fits the PCA basis on embeddings from both domains jointly. Combined with add_train_clips: true, this is a complete instance of feature-space domain alignment — and it accounts for the single largest observed OOF AUC improvement: +0.060 (0.7515 → 0.8041).
PCA explained variance by dimension — source-only vs. joint (train + soundscape)

Why Joint PCA Works as Domain Adaptation

A PCA fitted on source data only finds basis vectors that maximize variance in the source distribution. When target embeddings are projected into this basis, features that are discriminative in the target domain but low-variance in the source may be truncated by the PCA dimension cap.

By fitting PCA jointly on both domains, the basis vectors capture high-variance directions shared by both distributions. Target-specific discriminative features are preserved rather than discarded.

This is conceptually equivalent to shallow feature alignment methods in the DA literature (e.g., CORAL, CAN), implemented using the simplest possible tool: PCA.
