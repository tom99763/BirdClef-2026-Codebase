3. dual-foundation Notebook Updates dual-foundation-protossm-v3-sed-fusion.ipynb

Change 1 — LGBM Probe (replacing MLP)

The MLP probe was replaced by a per-class LGBMClassifier with scale_pos_weight to handle extreme class imbalance. One independent LGBM is trained per species (234 total), allowing each classifier to learn species-specific decision boundaries.

# Previous: single MLP over all classes
# New: per-class LGBM with imbalance handling
from lightgbm import LGBMClassifier

classifiers = {}
for i, species in enumerate(SPECIES_LIST):
    pos = labels[:, i].sum()
    neg = len(labels) - pos
    spw = neg / max(pos, 1)          # scale_pos_weight
    clf = LGBMClassifier(
        n_estimators=200,
        learning_rate=0.05,
        scale_pos_weight=spw,
        n_jobs=1,
        verbose=-1,
    )
    clf.fit(X_train[:, :, i].reshape(-1, 74), labels[:, i])
    classifiers[species] = clf

Features (74-dim): PCA(64) on 1536-d Perch embedding + 10 metadata features (hour bins, site priors, confidence stats).

Change 2 — TFLite Heads Blend (3 heads, alpha=0.30)

Three fine-tuned TFLite heads are blended additively at HEAD_BLEND_ALPHA=0.30. Each head was fine-tuned on Perch embeddings for a different supervision signal, providing complementary information to the LGBM probe.

HeadTraining DataBlend Weight
label_head_pseudoPseudo-labeled train_audio0.30
label_head_soundscape_trainLabeled soundscape recordings0.30
embedding_head_nohumanVAD-filtered nohuman data0.30
# Blend formula (applied after LGBM probe output)
blended = (
    (1 - HEAD_BLEND_ALPHA) * lgbm_probs
    + HEAD_BLEND_ALPHA / 3 * head1_probs
    + HEAD_BLEND_ALPHA / 3 * head2_probs
    + HEAD_BLEND_ALPHA / 3 * head3_probs
)

The three heads run in the CPU-only Kaggle notebook environment via TFLite interpreter. No GPU required at inference.

Change 3 — 2-way OOF Grid Search

The ensemble weight for ProtoSSM is determined via a 2-stage OOF grid search rather than a fixed value:

  1. Stage 1: Optimize LGBM probe alpha and TFLite heads blend alpha jointly on OOF folds.
  2. Stage 2: With stage-1 outputs fixed, grid-search ENSEMBLE_WEIGHT_PROTO (ProtoSSM vs. probe blend) on OOF.
# Stage 2: sweep ProtoSSM weight over OOF
best_auc, best_w = 0, 0.5
for w in np.arange(0.10, 0.91, 0.05):
    blend = w * proto_oof + (1-w) * probe_oof
    auc = macro_roc_auc(labels_oof, blend)
    if auc > best_auc:
        best_auc, best_w = auc, w
ENSEMBLE_WEIGHT_PROTO = best_w   # used in final test scoring

Change 4 — Global Seed Fixing

All sources of randomness are fixed at SEED=42 to ensure reproducible submissions across re-runs:

import random, numpy as np, torch

SEED = 42
random.seed(SEED)
np.random.seed(SEED)
torch.manual_seed(SEED)
torch.cuda.manual_seed_all(SEED)
torch.use_deterministic_algorithms(True)   # enforces CUDA determinism
torch.backends.cudnn.benchmark = False

Without this, LGBM model selection via OOF and ProtoSSM weight gridsearch could differ between runs, making it impossible to attribute LB changes to actual model improvements vs. random variation.

4. ResidualSSM Architecture & Position Analysis

ResidualSSM Architecture

The ResidualSSM is a small Bidirectional Selective SSM (BiSSM) that operates on the sequence of per-frame ProtoSSM probabilities and outputs an additive correction delta:

Input:  first_pass probs  [T × 234]
          (T = number of 5s frames in soundscape, 234 = species)

  → input_proj: Linear(234 → 128) + LayerNorm + GELU
  → BiSSM forward pass:  SSM(d_model=128, d_state=16, direction=fwd) → h_fwd [T × 128]
  → BiSSM backward pass: SSM(d_model=128, d_state=16, direction=bwd) → h_bwd [T × 128]
  → concat: [h_fwd || h_bwd]  [T × 256]
  → output_proj: Linear(256 → 234) + Tanh()  (bounded correction)
  → delta [T × 234]

Output: first_pass_corrected = first_pass + delta  [T × 234]
        clipped to [0, 1] after addition

The Tanh activation on the output layer bounds the correction to [-1, +1], preventing the corrector from producing degenerate outputs when applied to out-of-distribution inputs.

The Distribution Mismatch Problem

On 2026-03-24, applying the ResidualSSM correction after the VLOM blend caused LB to drop from 0.927 to 0.923. This was diagnosed as a distribution mismatch:

PositionInput DistributionResidualSSM Trained OnResult
After VLOM wrong 0.50×ProtoSSM + 0.50×SED: bimodal, blended statistics Pure ProtoSSM first_pass distribution LB 0.923 (-0.004)
Before VLOM correct Pure ProtoSSM first_pass probabilities Pure ProtoSSM first_pass distribution LB 0.927 (recovery)

The VLOM blend mixes SED ONNX predictions (different calibration, different mean probability) with ProtoSSM outputs. The ResidualSSM was trained on pure ProtoSSM distributions, so applying it to the blended output causes systematic miscorrection.

5. NS SED Residual Corrector (BiSSM)

Purpose and Motivation

After each NS SED round's inference on all soundscapes (all_ss_probs.npz), the SED student has systematic errors relative to the Perch teacher. Rather than feeding these raw errors directly into the pseudo label generator, we train a small BiSSM to learn the residual between teacher and student in probability space, then apply it to produce corrected predictions.

Training target: delta = teacher_probs - SED_R_probs
The BiSSM learns to predict this gap from the temporal context of SED predictions alone.

Architecture: train_sed_residual_corrector.py

# Input: SED probabilities for one soundscape (sequence of 5s frames)
SED_probs   [T × 234]   # from all_ss_probs.npz
teacher     [T × 234]   # from perch_teacher_aug_all_ss.csv (5s aligned)

# Model architecture
class SEDResidualCorrector(nn.Module):
    def __init__(self, n_species=234, d_model=128, d_state=16):
        self.input_proj  = nn.Sequential(
            nn.Linear(n_species, d_model),
            nn.LayerNorm(d_model),
            nn.GELU(),
        )
        self.ssm_fwd = SelectiveSSM(d_model, d_state)   # forward in time
        self.ssm_bwd = SelectiveSSM(d_model, d_state)   # backward in time
        self.output_proj = nn.Sequential(
            nn.Linear(d_model * 2, n_species),
            nn.Tanh(),                                   # bounded [-1, 1]
        )

    def forward(self, x):           # x: [B, T, 234]
        h = self.input_proj(x)       # [B, T, 128]
        hf = self.ssm_fwd(h)         # [B, T, 128]
        hb = self.ssm_bwd(h.flip(1)).flip(1)   # [B, T, 128]
        cat = torch.cat([hf, hb], dim=-1)  # [B, T, 256]
        return self.output_proj(cat)  # [B, T, 234] residual delta

# Training
loss = F.mse_loss(pred_delta, true_delta)   # true_delta = teacher - SED
corrected = (SED_probs + pred_delta).clamp(0, 1)

# Teacher weight schedule: R1=0.50, R2=0.30, R3=0.10
# Threshold pct schedule: R1=92, R2=93, R3=94
Why BiSSM for the corrector? The SED student's errors have temporal structure — it tends to miss species at the start/end of their calls and over-predict during silence. A BiSSM can leverage both past and future context within a soundscape to predict these systematic temporal errors. A simple per-frame correction (e.g., calibration layer) cannot capture this temporal dependency.
6. Pseudo Label Strategy Evolution

Previous Strategy (before 2026-03-24)

Pseudo labels for NS SED rounds were generated from SED student predictions only (no teacher retention after R0). The threshold percentile was fixed at 95 across all rounds.

RoundSourcesTeacher WeightThreshold Pct
R0Perch teacher only1.0095
R1SED student only0.0095
R2SED student only0.0095
R3SED student only0.0095

Problem: Without the teacher anchor, the student could progressively reinforce its own errors across rounds (confirmation bias / label drift).

Updated Strategy (2026-03-24)

Perch teacher is retained throughout all rounds at decreasing weight. The threshold percentile increases each round to accept only progressively more confident pseudo labels as the student improves.

RoundSourcesTeacher WeightThreshold PctRationale
R0 Perch teacher only 1.0095 Pure teacher; no student yet
R1 Corrected SED + Teacher 0.5092 Student bootstrapping; equal blend
R2 Corrected SED + Teacher 0.3093 Student improving; reduce teacher
R3 Corrected SED + Teacher 0.1094 Student dominant; teacher as anchor
R4 Inference only — no new pseudo labels Final round

1st-Place Insight: Teacher Retention Prevents Confirmation Bias

  Round N:   student errors → pseudo labels → Round N+1 training
                    ↑                                    |
                    └────────────────────────────────────┘
                         confirmation bias loop

  Fix: keep teacher_w > 0 at every round:
  pseudo_label = teacher_w * teacher_probs + (1-teacher_w) * corrected_SED_probs

The Perch teacher is a fixed, high-quality signal trained on labeled data. Even a small teacher_w=0.10 at R3 is enough to prevent the student from drifting into a degenerate solution where it confidently predicts the wrong species.

This was identified as a key technique from the BirdCLEF 2025 1st-place solution (Babych). Their ablation showed teacher dropout at any round caused noticeable AUC degradation.

Threshold Schedule Justification

Lower threshold (R1=92) accepts more pseudo labels, supporting learning when the student is still weak. As the student improves, stricter thresholds (R2=93, R3=94) ensure only high-confidence, high-quality labels are used, preventing overconfident noise from entering training.

  R1 (threshold=92nd pct): larger pseudo label set, more coverage, some noise OK
                  ↓
  R2 (threshold=93rd pct): pruned set, higher precision pseudo labels
                  ↓
  R3 (threshold=94th pct): tight set, highest precision, minimal noise
7. Lessons Learned
⚠️

Distribution Mismatch: Know Where Your Model Was Trained

The ResidualSSM was trained on pure ProtoSSM first_pass distributions. Applying it to VLOM-blended outputs (50% SED + 50% ProtoSSM) caused a -0.004 LB regression because the input distribution at inference time did not match training time. Always verify that the input distribution of a corrector/calibrator matches the distribution it was trained on. The fix was architecturally simple (move the corrector to before the blend), but cost one submission slot to discover.

🔑

LGBM Probe Superiority Over MLP for Rare Species

Per-class LGBMClassifier with scale_pos_weight outperforms a shared MLP because: (1) each species can have its own decision boundary without interference from other species; (2) scale_pos_weight directly handles the 1:500+ positive:negative ratio common in rare species; (3) LGBM handles feature interactions (embedding dims × metadata) without explicit engineering.

💻

CPU-Only Kaggle Environment Constrains Architecture

All TFLite heads and the ONNX SED model must run on CPU in the Kaggle notebook environment. This limits the corrector's complexity — the BiSSM corrector must be exportable to ONNX/TFLite and run within Kaggle's 9-hour time limit. The d_model=128 size was chosen to balance correction quality vs. inference time.

📈

2-Way OOF Outperforms Manual Weight Tuning

Prior ensemble weights (Perch vs. ProtoSSM) were set manually or via simple 1D grid search. The 2-way OOF approach jointly optimizes the LGBM/heads stage and the ProtoSSM stage, finding a better global optimum. The key insight is that these two stages interact: better LGBM outputs change the optimal ProtoSSM weight.

🔧

Pseudo Label Confirmation Bias Is a Real Risk

The 1st-place insight about teacher retention is not merely theoretical — it directly affects R2+ pseudo label quality. Without the teacher anchor, the corrected SED probs are still noisy at R1 (fold0=0.9433, fold1=0.9037, which implies the 5-fold ensemble is ~0.92 AUC on soundscapes), meaning ~8% error rate compounds across rounds. A 10% teacher anchor prevents this compounding.
