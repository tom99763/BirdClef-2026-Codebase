🏗 1. Model Architecture

Input : 20s waveform (640,000 samples @ 32 kHz) ↓ MelSpectrogram(n_mels=224, n_fft=2048, hop=512, fmin=0, fmax=16000, power=2, norm=slaney, mel_scale=htk) ↓ AmplitudeToDB(top_db=80) → normalize [0, 1] → expand to 3 channels ↓ ┌─────────────────────────────────────────────────────────────┐ │ EfficientNet-B0 (tf_efficientnet_b0.ns_jft_in1k) │ ← main branch │ drop_path_rate=0.15, no global pool, num_features=1280 │ │ — OR — │ │ PVT v2 B0 (pvt_v2_b0, 3.4M params) │ ← transformer branch │ Pyramid Vision Transformer, num_features=256 │ └─────────────────────────────────────────────────────────────┘ ↓ GEMFreqPool(p=3.0) [frequency axis only] → (B, C, T_time) ↓ AttentionSEDHead fc(C→C) + ReLU + Dropout att_conv(C→234, k=1) + cls_conv(C→234, k=1) output = sigmoid( softmax(tanh(att)) ⊙ cls ).sum(time_dim) ↓ Output : (B, 234) class probabilities
20-second clip duration — covers 4× Perch's 5s windows, forcing the model to learn soundscape-domain temporal context. Core insight from the 1st-place solution.
GEMFreqPool — Generalized Mean Pooling (p=3.0) on the frequency axis only. Time axis is preserved and handled by the attention head, enabling per-window temporal detection.

🗄 2. Training Data Design — Domain Adaptation via 3-Source Mixing

The key insight: BirdCLEF test data is soundscape recordings (60s, multi-species, background noise), but most labeled training data is point recordings (close-up, single species). Noisy Student bridges this domain gap by training on all three sources simultaneously.

① train_audio/ — Real Labels
~24,000 clips
Point recordings with ground-truth labels from train.csv.
All data used every round — no fold split on this source.
Domain: close-up recordings, single species, clean audio.
+
② train_soundscapes/ — Real Labels
~1,479 windows (5s each)
Human-annotated labels from train_soundscapes_labels.csv.
Split by GroupKFold: validation files held out per fold; remaining files enter training.
Domain: same as test — soundscape recordings.
+
③ train_soundscapes/ — Pseudo Labels
~119,000 windows (Round 6)
Model-generated soft labels from pseudo_labels/sed_20s_r{R}.csv.
Covers soundscape files without human annotation, greatly expanding coverage.
Validation files automatically excluded each fold. Grows in quality each round.
↓
Cross-Domain MixUp (λ=0.5)
Each labeled train_audio clip is mixed 1-to-1 with a pseudo soundscape clip at λ=0.5.
Forces the model to jointly learn point-recording and soundscape features in every batch.
Why this works: The competition test set is soundscape recordings. By iteratively generating pseudo labels on soundscapes and mixing them into training, each NS round pulls the model's representations closer to the test domain.
Val split design: train_soundscapes_labels.csv serves a dual role — the held-out files form the validation set (ss_auc), while the remaining files provide real-labeled training data. Pseudo labels from pseudo_labels_csv are complementary: they cover the ~580 soundscape files that have no human annotation.
nonaves_perch_only: SED models are unreliable for non-bird classes (Amphibia, Insecta, Mammalia, Reptilia) due to class imbalance. For these classes, pseudo labels always come from Perch predictions only — preventing noise from propagating across rounds.

🔬 7. PVT v2 B0 — Independent Transformer Branch

Tests whether a Pyramid Vision Transformer backbone outperforms EfficientNet-B0 CNN. Audio is processed as a 2D mel spectrogram image, making any vision backbone applicable. PVT v2 B0 (3.4M params) has richer long-range dependency modeling via self-attention across frequency and time.

Independent by design: PVT starts from B0 R4 pseudo labels and produces its own sed_20s_pvt_r{N}.csv chain independently — does not block the B0 pipeline.
Gap closing: R1 gap vs B0 = −0.012, R2 gap = −0.003, R3 gap ≈ −0.005 (4/5 folds). Architecture diversity benefit in ensemble expected even if PVT doesn't individually surpass B0.

🎯 8. Inference: 20s Sliding Window (Window Aggregation)

Problem solved: Training uses 20s clips but competition expects 5s slot predictions (12 slots per 60s file). Naïve 5s inference creates a train/test mismatch.
60s audio file ↓ build 9 overlapping 20s windows (stride = 5s) ↓ window 0: samples [ 0 → 640000] covers slots 0, 1, 2, 3 window 1: samples [ 160000 → 800000] covers slots 1, 2, 3, 4 window 2: samples [ 320000 → 960000] covers slots 2, 3, 4, 5 ... window 8: samples [1280000 →1920000] covers slots 8, 9,10,11 ↓ SED ONNX inference on all 9 windows → (9, 234) probs ↓ Accumulate with overlap counting: slot s ← average of all windows covering that slot edge slots (0, 11): covered by 1 window center slots (3–8): covered by 4 windows ↓ (12, 234) per-slot predictions ↓ BranchEns → cSEBBs post-processing → final submission
Confirmed LB 0.931 with window aggregation approach. Framewise inference (true 1st-place method) was tested and scored only LB 0.924 with our model — window aggregation is better for our architecture.
