🔊 CLAP Cross-Domain Contrastive Bridge
BirdCLEF 2026 Experiment Design — 2026-03-30

Motivation

The core challenge of BirdCLEF 2026 is domain gap: training data (train_audio) consists of clean close-range recordings, while evaluation data (soundscapes) are complex wild-field recordings with multiple overlapping species, background noise, and distant sources.

train_audio: clean vocalizations, single species, high SNR, studio-like quality soundscape: multi-species overlap, low SNR, distant recording, background noise → 66.8% Amphibia, 32.2% Aves → current best val AUC ~0.919 (SED + Perch ensemble)

Goal: train a domain-invariant audio encoder so that embeddings of the same species in clean vs. wild domains are as close as possible in latent space.

1.3 InfoNCE Loss

L = (1/2N) × Σᵢ [
    -log( exp(zᵢᵃ · zᵢᵗ / τ) / Σⱼ exp(zᵢᵃ · zⱼᵗ / τ) )   # audio → text
    -log( exp(zᵢᵗ · zᵢᵃ / τ) / Σⱼ exp(zᵢᵗ · zⱼᵃ / τ) )   # text → audio
]
τ = learnable temperature (init: 1/0.07 ≈ 14.3)

2. Core Innovation: Audio-Audio Cross-Domain Contrastive

Key Insight: Replace audio-text pairs with audio-audio cross-domain pairs. Same species in clean domain (train_audio) and wild domain (soundscape pseudo clip) form a positive pair. No text encoder needed during training — only the audio encoder is fine-tuned.

2.1 Positive / Negative Pair Definition

Positive pair (same species, different domain): anchor: train_audio/Thraupis_sayaca/XC123456.ogg (clean, Thraupis sayaca) positive: soundscape clip @ t=25s (wild, pseudo label: Thraupis sayaca, conf > 0.8) Negative pair (different species — in-batch negatives): train_audio/Furnarius_rufus/XC789.ogg (different species) any soundscape clip of a different species

2.2 Full Architecture

Design Principle: CLAP is purely a frozen feature extractor. Both the audio encoder (HTSAT, 27.5M) and text encoder (RoBERTa, 124.6M) are completely frozen — eval() + torch.no_grad() at all times. Only the lightweight projection head and classifier are trained. CLAP features can be pre-extracted and cached to disk for fast training.
CLAP Audio Encoder (HTSAT, 27.5M) — FULLY FROZEN, eval+no_grad ↓ 512-dim audio embedding (pre-computed & cached) ════════════════════════════════════════════════════════ TRAINABLE COMPONENTS ONLY: ┌─────────────────────────────────────────────────────┐ │ Projection Head (512 → 256) │ │ BatchNorm → ReLU → Linear → L2 norm │ │ ~200K params — cross-domain SupCon │ └──────────────────────┬──────────────────────────────┘ │ ┌──────────────────────▼──────────────────────────────┐ │ Classification Head (512 → 234) │ │ Dropout(0.1) → Linear │ │ ~120K params — FocalBCE │ └──────────────────────┬──────────────────────────────┘ │ Loss = λ₁ × SupConLoss(proj, labels) + λ₂ × FocalBCE(logits, labels)
Benefits of frozen CLAP: (1) No risk of destroying pretrained features. (2) Pre-compute all embeddings once → training is just MLP forward passes, very fast. (3) Much lower GPU memory. (4) No 48 kHz resample overhead at training time (only at feature extraction time).

2.3 SupConLoss (Supervised Contrastive)

# Stronger than SimCLR: uses species labels to define all positives
L_SupCon = Σᵢ (-1/|P(i)|) × Σ_{p∈P(i)} log[
    exp(zᵢ · z_p / τ) / Σ_{k≠i} exp(zᵢ · z_k / τ)
]
# P(i) = all samples with the same species as i (across both domains)
# τ = 0.07 (fixed or learnable)

3.3 Prompt Template

def build_text_prompt(row, taxonomy_df):
    sp = taxonomy_df.loc[row.primary_label]

    # 1. Primary species (always present)
    prompt = (f"sound of {sp['common_name']} "
              f"({sp['scientific_name']}, {sp['class_name']})")

    # 2. Call type (if available and not 'uncertain')
    if row['type'] not in ['[]', "['uncertain']", '']:
        call_type = eval(row['type'])[0]
        prompt += f", {call_type}"

    # 3. Secondary species (up to 2)
    if row['secondary_labels'] != '[]':
        secs = eval(row['secondary_labels'])
        sec_names = [taxonomy_df.loc[s]['common_name']
                     for s in secs[:2] if s in taxonomy_df.index]
        if sec_names:
            prompt += f", with background sounds of {' and '.join(sec_names)}"

    # 4. Location anchor (fixed)
    prompt += ", recorded in Pantanal, Brazil"

    return prompt

3.4 Soundscape Text Prompt (Wild Domain)

def build_soundscape_prompt(pseudo_scores, taxonomy_df, conf_thr=0.8):
    """Build text from pseudo labels for a soundscape window."""
    top = pseudo_scores.nlargest(3)
    primary = top.index[0]
    sp = taxonomy_df.loc[primary]

    prompt = (f"wild soundscape recording containing "
              f"{sp['common_name']} ({sp['scientific_name']})")

    others = [taxonomy_df.loc[s]['common_name']
              for s in top.index[1:] if top[s] > conf_thr
              and s in taxonomy_df.index]
    if others:
        prompt += f" and {', '.join(others)}"

    prompt += ", in Pantanal, Brazil wetland environment"
    return prompt

# Example:
# "wild soundscape recording containing Sayaca Tanager (Thraupis sayaca)
#  and Rufous Hornero, in Pantanal, Brazil wetland environment"
Key Design Choice: Clean and wild text prompts are intentionally written differently — clean: "clear recording of {species}", wild: "wild soundscape containing {species}". This teaches the text encoder the domain distinction while forcing the audio encoder to learn domain-invariant species representations.

5. Training Strategy

5.1 Two-Stage Training

Stage 1 — Contrastive Pre-training (2–3 weeks)
  • Load HTSAT from laion/clap-htsat-unfused
  • Text encoder: eval() + torch.no_grad() always
  • HTSAT stages 0–1: frozen; stages 2–3 + projection head: fine-tuned
  • Loss = SupConLoss (cross-domain)
  • LR = 1e-4, warmup 500 steps, cosine decay
  • Batch size = 128 (large batch critical for contrastive)
  • Input: 32 kHz → resample to 48 kHz → ClapProcessor → HTSAT
Stage 2 — Classification Fine-tuning (1–2 weeks)
  • Add classification head (768 → 234) on top of Stage 1 checkpoint
  • Loss = FocalBCE (primary) + SupConLoss × 0.1 (auxiliary, preserves domain invariance)
  • LR = 5e-5 (smaller to avoid destroying contrastive features)
  • Use full BirdCLEF train_audio + R4 pseudo labels
Inference
  • Stage 2 fine-tuned model in eval() + no_grad()
  • Sliding window: 20 s → 5 s per window (4 overlapping chunks, same as SED NS)
  • 5-fold ensemble

5.2 Loss Weights

L_total = L_FocalBCE + λ_con × L_SupCon

Stage 1: λ_con = 1.0  (pure contrastive)
Stage 2: λ_con = 0.1  (regularizer during fine-tuning)

8.1 CLAP Feature Extractor (Frozen) + Trainable Head

from transformers import ClapModel, ClapProcessor
import torch, torch.nn as nn, torch.nn.functional as F

# ── Step 1: Frozen CLAP feature extractor ────────────────────────────────
class ClapExtractor:
    """Frozen CLAP — always eval() + no_grad(). Never call .train()."""
    def __init__(self, pretrained="laion/clap-htsat-unfused", device="cuda"):
        clap = ClapModel.from_pretrained(pretrained).to(device)
        clap.eval()
        for p in clap.parameters():
            p.requires_grad = False
        self.audio_model      = clap.audio_model
        self.audio_projection = clap.audio_projection
        self.text_model       = clap.text_model
        self.text_projection  = clap.text_projection
        self.processor        = ClapProcessor.from_pretrained(pretrained)
        self.device           = device

    @torch.no_grad()
    def extract_audio(self, wav_48k_np: np.ndarray) -> torch.Tensor:
        """wav_48k_np: (T,) numpy float32 at 48 kHz → (512,) embedding."""
        inputs = self.processor(audios=wav_48k_np, sampling_rate=48000,
                                return_tensors="pt")
        feat = inputs['input_features'].to(self.device)
        out  = self.audio_model(input_features=feat)
        emb  = self.audio_projection(out.pooler_output)  # (1, 512)
        return F.normalize(emb, dim=-1).squeeze(0)       # (512,)

    @torch.no_grad()
    def extract_text(self, text: str) -> torch.Tensor:
        """text → (512,) embedding."""
        inputs = self.processor(text=text, return_tensors="pt",
                                padding=True, truncation=True)
        inputs = {k: v.to(self.device) for k, v in inputs.items()}
        out  = self.text_model(**inputs)
        emb  = self.text_projection(out.pooler_output)  # (1, 512)
        return F.normalize(emb, dim=-1).squeeze(0)      # (512,)


# ── Step 2: Trainable head (only this is updated during training) ─────────
class ClapBirdHead(nn.Module):
    """Lightweight trainable head on top of frozen CLAP embeddings."""
    def __init__(self, embed_dim=512, proj_dim=256,
                 num_classes=234, dropout=0.1):
        super().__init__()
        # Projection head for SupConLoss
        self.proj = nn.Sequential(
            nn.Linear(embed_dim, proj_dim),
            nn.BatchNorm1d(proj_dim),
            nn.ReLU(inplace=True),
            nn.Linear(proj_dim, proj_dim),
        )
        # Classification head for FocalBCE
        self.classifier = nn.Sequential(
            nn.Dropout(dropout),
            nn.Linear(embed_dim, num_classes),
        )

    def forward(self, emb: torch.Tensor):
        """emb: (B, 512) pre-computed frozen CLAP embedding."""
        logits = self.classifier(emb)
        proj   = F.normalize(self.proj(emb), dim=-1)
        return {'logits': logits, 'proj': proj}

8.2 Pre-compute and Cache CLAP Embeddings

import torchaudio, numpy as np
from pathlib import Path

def precompute_clap_embeddings(audio_files, extractor, cache_dir,
                               sr_in=32000, sr_out=48000, clip_dur=5):
    """
    Pre-extract CLAP embeddings for all audio files and cache to disk.
    Only needs to run ONCE — CLAP is frozen so embeddings never change.
    """
    cache_dir = Path(cache_dir)
    cache_dir.mkdir(parents=True, exist_ok=True)

    for fpath in audio_files:
        cache_path = cache_dir / (Path(fpath).stem + ".npy")
        if cache_path.exists():
            continue
        wav, sr = torchaudio.load(fpath)
        if sr != sr_out:
            wav = torchaudio.functional.resample(wav, sr, sr_out)
        # Random crop to clip_dur seconds
        max_start = max(0, wav.shape[1] - sr_out * clip_dur)
        start = np.random.randint(0, max_start + 1)
        clip = wav[0, start:start + sr_out * clip_dur].numpy()
        emb = extractor.extract_audio(clip)  # (512,)
        np.save(cache_path, emb.cpu().numpy())

8.3 CrossDomainEmbeddingDataset (loads pre-cached embeddings)

class CrossDomainEmbeddingDataset(Dataset):
    """
    Loads pre-computed CLAP embeddings from disk.
    Training is just MLP forward passes — very fast.
    Each item: (clean_emb, wild_emb, label)
    """
    def __init__(self, clean_emb_dir, wild_emb_dir,
                 clean_meta, wild_meta, taxonomy_df):
        # clean_meta: DataFrame with columns [filename, primary_label]
        # wild_meta:  DataFrame with columns [cache_file, primary_label, conf]
        self.clean_dir = Path(clean_emb_dir)
        self.wild_dir  = Path(wild_emb_dir)

        # Build per-species lists of cached embedding files
        self.clean = defaultdict(list)
        for _, row in clean_meta.iterrows():
            stem = Path(row.filename).stem
            p = self.clean_dir / f"{stem}.npy"
            if p.exists():
                self.clean[row.primary_label].append(p)

        self.wild = defaultdict(list)
        for _, row in wild_meta[wild_meta.conf >= 0.8].iterrows():
            p = self.wild_dir / row.cache_file
            if p.exists():
                self.wild[row.primary_label].append(p)

        # Only species with both clean and wild embeddings
        self.species = sorted(set(self.clean) & set(self.wild))
        self.sp2idx  = {sp: i for i, sp in enumerate(self.species)}
        print(f"CrossDomainDataset: {len(self.species)} species, "
              f"{sum(len(v) for v in self.clean.values())} clean, "
              f"{sum(len(v) for v in self.wild.values())} wild embeddings")

    def __len__(self):
        return len(self.species) * 8  # oversample

    def __getitem__(self, idx):
        sp = self.species[idx % len(self.species)]
        clean_emb = np.load(random.choice(self.clean[sp]))  # (512,)
        wild_emb  = np.load(random.choice(self.wild[sp]))   # (512,)
        label = self.sp2idx[sp]
        return (torch.from_numpy(clean_emb).float(),
                torch.from_numpy(wild_emb).float(),
                label)

8.4 SupConLoss (Cross-Domain)

class CrossDomainSupConLoss(nn.Module):
    def __init__(self, temperature=0.07):
        super().__init__()
        self.temp = temperature

    def forward(self, z_clean, z_wild, labels):
        """
        z_clean, z_wild: (B, D) L2-normalized embeddings
        labels: (B,) species indices
        """
        # Concatenate both domains: (2B, D)
        z = torch.cat([z_clean, z_wild], dim=0)
        labels_2x = labels.repeat(2)

        # Similarity matrix (2B, 2B)
        sim = torch.mm(z, z.T) / self.temp
        sim.fill_diagonal_(-1e9)  # exclude self

        # Positive mask: same species (cross-domain included)
        pos_mask = (labels_2x.unsqueeze(0) == labels_2x.unsqueeze(1))
        pos_mask.fill_diagonal_(False)

        # SupCon loss
        log_prob = sim - torch.logsumexp(sim, dim=1, keepdim=True)
        n_pos = pos_mask.sum(1).clamp(min=1).float()
        loss = -(log_prob * pos_mask).sum(1) / n_pos
        return loss.mean()

9. Expected Outcomes and Risks

✅ Expected Outcomes

  • Domain-invariant embedding: clean/wild same-species closer in latent space
  • Soundscape val AUC improvement (+0.01 ~ +0.03 estimated)
  • Larger gain for rare species (when pseudo label quality is sufficient)
  • Complementary to SED NS + Perch ensemble (different feature space)
  • 3-model ensemble target: LB > 0.935

⚠ Key Risks

  • Noisy pseudo labels → noisy positive pairs → weak contrastive signal
  • 48 kHz resample overhead at training time (precompute recommended)
  • Large batch size requirement (128+) — GPU memory pressure
  • SupCon loss tuning (temperature, λ) adds complexity
  • HTSAT fine-tuning may overfit on small BirdCLEF dataset
